Claude Code 2.1.207 · Auto Mode GA auf Cloud-Plattformen
Deutsch

Claude Code 2.1.207 macht Auto Mode zur neuen standardmäßigen Einstellung auf allen drei großen Cloud-Plattformen: Bedrock, Vertex AI und Foundry — wer ihn deaktiviert, setzt disableAutoMode: true in der ~/.claude/settings.json. Gleichzeitig wechseln alle drei Plattformen als Default-Modell auf Claude Opus 4.8, was in bestehenden Deployments ein aufmerksames Auge auf die Kosten erfordert. Dazu gibt es eine neue Sicherheitsregel, die verhindert, dass Auto Mode die Session-Transcripts manipuliert. Wer die Einstellung bisher über settings.local.json gesteuert hat, muss auf ~/.claude/settings.json wechseln — der settings.local.json-Pfad wird ab jetzt ignoriert.

English

Claude Code 2.1.207 makes Auto Mode the new default on all three major cloud platforms — Bedrock, Vertex AI, and Foundry. To opt out, set disableAutoMode: true in ~/.claude/settings.json. Simultaneously all three platforms switch their default model to Claude Opus 4.8, which means existing deployments should watch costs carefully. A new security rule blocks Auto Mode from tampering with session transcript files. Anyone who was managing this setting through settings.local.json needs to migrate to ~/.claude/settings.json — that path is no longer read.

Deutsch

English

Mistral has released Robostral Navigate, its first model for embodied AI — an 8B model that teaches robots to navigate unknown environments autonomously, using only a single RGB camera with no LiDAR or depth sensors required. The model takes an image and a plain-language command such as "walk through the corridor into the supply room and stop in front of the second shelf," then outputs navigation commands directly. It achieves a 76.6% success rate on the R2R-CE benchmark in unseen environments, trained entirely in simulation on 400,000 trajectories across 6,000 unique scenes. For logistics, manufacturing, and delivery, it marks a meaningful entry point into physical AI.

Claude Code 2.1.202 · /review vs. /code-review: zwei Befehle, zwei Modi
Deutsch

Mit v2.1.202 zieht Claude Code eine saubere Trennlinie zwischen zwei Review-Befehlen: Der eine macht einen schnellen Einzeldurchlauf — gut für Quick-Checks während der Entwicklung. Der andere schickt mehrere Agenten mit wählbarem Effort-Level in den Ring: low, medium, high oder max. Wer gründliche Reviews mit mehreren Perspektiven will — etwa bevor ein Branch auf main landet — greift zum zweiten Befehl und wählt den Effort je nach Criticality. Außerdem gibt es ab jetzt im Config-Menü eine neue Einstellung, die den Agent-Count in Ultracode-Workflows auf klein, mittel oder groß reguliert, damit du Token-Verbrauch und Laufzeit gezielt gegeneinander abwägst.

English

With v2.1.202, Claude Code draws a clean line between two review commands: one does a fast single pass — good for quick checks during development. The other sends multiple agents in with a selectable effort level: low, medium, high, or max. For thorough, multi-perspective reviews before a branch lands on main, use the second command and pick the effort level according to criticality. There's also a new "Dynamic workflow size" option in the config menu that dials the agent count in ultracode workflows to small, medium, or large, so you can consciously trade off token spend against wall-clock time.
