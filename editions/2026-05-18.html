💡 Hack des Tages · Tip of the day

Claude Code 2.1.140 · Plugin-Konflikte sichtbar in /doctor
Deutsch

Claude Code 2.1.140 löst einen häufigen Silent-Fail in Plugins: Wenn du in der plugin.json einen Ordner wie commands explizit setzt, wird der gleichnamige Standardordner still ignoriert — und deine Befehle tauchen nirgendwo auf. Ab 2.1.140 werden diese Konflikte explizit in /doctor, beim Slash-Command /plugin und beim Aufruf claude plugin list angezeigt. Wer also Plugin-Befehle vermisst, sollte als Erstes dort nachschauen — das spart langes Rätselraten.

English

Claude Code 2.1.140 surfaces a common silent failure in plugins: if you explicitly set a folder like commands in plugin.json, any default folder with the same name is quietly ignored — and your commands vanish. From 2.1.140, these conflicts are flagged explicitly in /doctor, in the /plugin slash command, and when running claude plugin list. If your plugin commands are missing, that's the first place to check — saves a lot of head-scratching.

BEFEHL · COMMAND
/doctor
claude plugin list
# Plugin-Konfigurationskonflikte sehen · Spot plugin.json conflicts
$ /doctor

# Alle geladenen Komponenten eines Plugins auflisten · List a plugin's loaded components
$ claude plugin list
