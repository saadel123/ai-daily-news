Malware-Slop · npm stiehlt Claude-Nutzerdaten
Deutsch

Sicherheitsforscher haben ein bösartiges npm-Paket namens «mouse5212-super-formatter» entdeckt, das gezielt das Claude-AI-Benutzerverzeichnis ausspioniert — dort wo Claude Code Uploads und Outputs ablegt. Das Paket schleust sich beim Installieren ein, holt sich einen GitHub-Token aus der Umgebung und lädt alle Workspace-Dateien in ein Repository des Angreifers hoch. Der irre Twist: Der Angreifer hat seinen eigenen privaten GitHub-Token im Paketcode eingebettet und damit sein gesamtes Konto preisgegeben. Das Paket wurde rund 676-mal heruntergeladen, bevor es entfernt wurde — eine Erinnerung, npm-Abhängigkeiten vor dem Einbinden zu prüfen.

English

Security researchers uncovered a malicious npm package called "mouse5212-super-formatter" that specifically targeted the Claude AI user directory — where Claude Code stores its uploads and outputs. The package fires a postinstall script that grabs a GitHub token from the environment and uploads all workspace files to an attacker-controlled repository. The ironic twist: the attacker hard-coded their own private GitHub token in the package, exposing their entire account. The package was downloaded around 676 times before removal — a reminder to audit npm dependencies before bringing them in.

💡 Hack des Tages · Tip of the day

Claude Code 2.1.154 · ! Shell-Befehl als Hintergrundsession
Deutsch

In der Agents-Ansicht von Claude Code gibt es seit v2.1.154 einen praktischen Trick: Mit einem Ausrufezeichen gefolgt vom Befehl startet Claude Code den Shell-Befehl als eigenständige Hintergrundsession — mit eigenem Status-Eintrag, den du wie jeden anderen Agenten an- und abkoppeln kannst. Ideal, wenn du einen lang laufenden Build oder eine Test-Suite im Blick behalten willst, während du in anderen Sessions weitermachst. Alternativ startest du die gleiche Session direkt beim Launch von Claude Code über ein entsprechendes Flag.

English

Since v2.1.154, the claude agents view has a handy trick: type an exclamation mark followed by a shell command and Claude Code launches it as an independent background session — with its own status entry you can attach to and detach from just like any other agent. Perfect for keeping an eye on a long-running build or test suite while you continue work in other sessions. The same behavior is also available as a startup flag when launching Claude Code directly.

BEFEHL · COMMAND
! <shell-command>
claude --bg --exec '<shell-command>'
--bg
start as a detached background session visible in the agents view
--exec
shell command to run as the background session
# Im Agents-View einen Build als Hintergrundsession starten · Start a build as background session
$ ! npm run build

# Direkt beim Launch als Hintergrundsession starten · Launch as background session at startup
$ claude --bg --exec 'npm test -- --watch'
