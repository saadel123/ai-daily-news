The new design introduces two dedicated local accounts: CodexSandboxOffline for default operation with all outbound traffic blocked at the OS level, and CodexSandboxOnline for when users explicitly authorize broader connectivity. The solution combines Windows ACLs, restricted tokens, and firewall rules without breaking ordinary development workflows.

Claude Code 2.1.142 · agents flags
Deutsch

Claude Code bekommt mit Version 2.1.142 acht neue Flags für den Dispatch-Befehl, der Hintergrund-Agenten startet — du kannst jetzt direkt beim Start das Arbeitsverzeichnis, die Settings-Datei, den MCP-Server, das Plugin-Verzeichnis, den Permission-Mode, das Modell und den Effort-Level festlegen. Außerdem läuft der Fast Mode ab sofort per Default auf Opus 4.7 statt Opus 4.6 — für alle, die schnellere Ausgaben brauchen, ohne das Modell manuell zu wechseln. Und wer ein Plugin mit einer SKILL.md im Root-Ordner hat, muss kein eigenes Skills-Unterverzeichnis mehr anlegen: Claude Code erkennt es automatisch als Skill.

English

Claude Code 2.1.142 ships eight new flags for the command that dispatches background agents — you can now set the working directory, settings file, MCP config, plugin directory, permission mode, model, and effort level directly at dispatch time, without editing a config file first. Fast mode now defaults to Opus 4.7 instead of Opus 4.6, so you get faster output without a manual model switch. And if your plugin has a SKILL.md in its root with no skills/ subdirectory, Claude Code now surfaces it as a skill automatically.

BEFEHL · COMMAND
claude agents [options]
--add-dir <path>
add a working directory to the dispatched background session
--settings <file>
use a specific settings file for the background session
--mcp-config <file>
load a specific MCP config into the dispatched session
--plugin-dir <path>
add a plugin directory to the dispatched background session
--permission-mode <mode>
set the permission mode (e.g. auto, ask) for the background session
--model <id>
override the model for this dispatched background session
--effort <level>
set the effort level: low, normal, high, xhigh, or max
--dangerously-skip-permissions
skip all permission prompts in the background session
# Agent mit Opus 4.7, xhigh-Effort und eigenem MCP-Config · Agent with Opus 4.7, xhigh effort, custom MCP
$ claude agents --model claude-opus-4-7 --effort xhigh --mcp-config ./mcp-prod.json

# Background-Agent im Auto-Mode starten · Dispatch a background agent in auto-mode
$ claude agents --permission-mode auto --add-dir ./src
