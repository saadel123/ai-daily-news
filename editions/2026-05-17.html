💡 Hack des Tages · Tip of the day

Claude Code 2.1.143 · Worktree-Isolation abschalten
Deutsch

Claude Code 2.1.143 bringt eine nützliche Einstellung für alle, die Background-Sessions in Repos nutzen, wo Git-Worktrees unpraktisch sind — zum Beispiel in Monorepos, auf Netzlaufwerken oder bei komplexen Submodule-Setups. Mit dem neuen Wert none für die Worktree-Hintergrund-Isolation arbeiten Background-Sessions direkt im Working-Copy, ohne einen separaten Worktree anzulegen. Das spart Disk-Space und umgeht Umgebungen, bei denen der Worktree-Layer schlicht nicht funktioniert. Einfach in die .claude/settings.json eintragen — und Background-Agents laufen, als wäre gar kein Isolation-Layer da.

English

Claude Code 2.1.143 adds a handy setting for teams running background sessions in repos where Git worktrees are impractical — think monorepos, network-mounted drives, or complex submodule setups. Setting the worktree background isolation to none makes background sessions work directly in the working copy, skipping the separate worktree entirely. It saves disk space and sidesteps environments where the worktree layer simply does not work. Drop it in your .claude/settings.json and background agents run as if there is no isolation layer at all.
