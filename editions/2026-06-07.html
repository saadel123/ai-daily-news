ChatGPT Lockdown Mode · Prompt Injection Defense
Deutsch

OpenAI hat den Lockdown Mode für ChatGPT eingeführt — ein Sicherheitsschalter, der beim Aktivieren Web-Browsing, Agent Mode, Deep Research, File Downloads, Canvas-Networking und Live-Connectors deaktiviert. Das Ziel: ausgehende Netzwerkanfragen bei Prompt-Injection-Angriffen unterbinden, bevor sensible Daten abfließen können. Der Modus rollt für alle Account-Typen aus — Free, Go, Plus, Pro und Business. Wichtig: Lockdown Mode verhindert nicht das Einschleusen von Prompt Injections — er sichert nur den Exfiltrations-Pfad ab.

English

OpenAI launched Lockdown Mode for ChatGPT — a security toggle that disables live web browsing, agent mode, deep research, file downloads, canvas networking, and live connectors when enabled. The goal: cut off outbound network requests before sensitive data can leak through a prompt-injection attack. The mode rolls out to all account types — Free, Go, Plus, Pro, and Business. Important caveat: Lockdown Mode does not stop prompt injections from appearing in content — it only secures the exfiltration path.

💡 Hack des Tages · Tip of the day

Claude Code 2.1.162 · agents JSON · Autocomplete fills
Deutsch

Claude Code 2.1.162 bringt gleich zwei praktische Quality-of-Life-Änderungen. Erstens: Wenn du im Slash-Command-Autocomplete einen Eintrag anklickst, wird er jetzt ins Eingabefeld eingetippt statt sofort ausgeführt — kein versehentliches Triggern mehr, du drückst erst Enter, wenn du wirklich bereit bist. Zweitens: die JSON-Ausgabe von claude agents enthält jetzt ein neues Feld, das zeigt, worauf eine wartende Session gerade blockiert ist — zum Beispiel ein Permission-Prompt. English

Claude Code 2.1.162 ships two quality-of-life improvements. First: clicking a slash command in the autocomplete menu now fills it into your input instead of running it immediately — no more accidental triggers; press Enter when you are actually ready. Second: the JSON output of claude agents now includes a new field showing what a waiting session is blocked on — for example, a permission prompt. For scripts that monitor multiple background agents this is a genuine win: see at a glance which agent is waiting for approval.

BEFEHL · COMMAND
claude agents --json
--json
output all agent sessions as JSON, including the new waitingFor field
# Welcher Agent wartet auf Freigabe? · Which agent is waiting for approval?
$ claude agents --json | jq '.[] | select(.waitingFor != null) | {name, waitingFor}'
