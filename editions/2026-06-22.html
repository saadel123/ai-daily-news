Anthropic · Workload Identity Federation GA
Deutsch

Anthropic hat Workload Identity Federation (WIF) für die Claude Platform in die General Availability gehoben — ab sofort kein statischer API-Key mehr nötig. Stattdessen authentifizieren sich Workloads mit dem Identity, das sie sowieso schon haben: eine AWS-IAM-Role, einen GCP-Service-Account, eine Azure-Managed-Identity, ein GitHub-Actions-Token oder einen anderen OIDC-kompatiblen Provider. Die kurzlebigen Credentials laufen automatisch ab, ohne manuelles Widerrufen. Dazu kommen Service Accounts, die jedem Workload einen eigenen Audit-Trail geben — WIF gilt für alle Claude-API-Endpoints, inklusive SDKs und Claude Code.

English

Anthropic has moved Workload Identity Federation (WIF) to general availability on the Claude Platform — no more static API keys required. Workloads now authenticate with the identity they already carry: an AWS IAM role, GCP service account, Azure managed identity, GitHub Actions token, or any OIDC-compliant provider. Short-lived credentials expire automatically, with no manual revocation needed. Service accounts give each workload its own identity, roles, and audit trail — WIF covers all Claude API endpoints, including the first-party SDKs and Claude Code.

Agentjacking · Sentry MCP · Claude Code, Cursor, Codex
Deutsch

Sicherheitsforscher von Tenet Security haben einen Angriff namens Agentjacking enthüllt, der einen architektonischen Fehler in Sentrys MCP-Integration ausnutzt: Ein Angreifer findet den öffentlichen DSN eines Projekts — oft im Frontend-Code sichtbar — und injiziert einen bösartigen Error-Event, den KI-Coding-Agenten wie Claude Code, Cursor oder Codex als legitime Remediation-Anweisung lesen. Die Erfolgsquote lag bei 85 % über alle getesteten Agenten, mit 2.388 betroffenen Organisationen. Sentry hat das Problem anerkannt, aber nicht an der Wurzel behoben — Entwickler sollten ihren Agenten daher keine Sentry-Fehler autonom abarbeiten lassen.

English

Security researchers at Tenet Security disclosed an attack called Agentjacking that exploits an architectural flaw in Sentry's MCP integration: an attacker gets hold of a project's public DSN — often visible in frontend code — and injects a malicious error event that AI coding agents like Claude Code, Cursor, or Codex read as legitimate remediation guidance. The success rate was 85% across all tested agents, with 2,388 organizations found to be exposed. Sentry acknowledged the issue but declined to fix it at the root — developers should avoid letting their agents autonomously action Sentry errors.

Claude Code 2.1.183 · attribution.sessionUrl · /config --help
Deutsch

Mit Version 2.1.183 hat Claude Code eine neue Einstellung, die den claude.ai-Session-Link aus Commits und Pull Requests herausnimmt — praktisch, wenn du im Browser oder über Remote Control arbeitest und nicht möchtest, dass interne Session-URLs ins Repository wandern. Bonus aus demselben Release: /config --help zeigt dir alle verfügbaren Shorthand-Keys auf einen Blick.

English

Claude Code 2.1.183 ships a setting that strips the claude.ai session link from commits and pull requests — handy when you work in the browser or via Remote Control and don't want internal session URLs leaking into the repository. You can set it on the fly directly from the prompt without touching settings.json. A neat bonus from the same release: /config --help lists every available shorthand key at a glance.
