💡 Hack des Tages · Tip of the day

Claude Code 2.1.174–175 · /usage Attribution
Deutsch

Claude Code 2.1.174 bringt für VSCode-Nutzer eine echte Verbesserung im Account-&-Usage-Dialog: Die Ansicht schlüsselt jetzt auf, woher deine Token-Kosten wirklich kommen — Cache-Misses, Long-Context-Nutzung, Subagenten und, besonders praktisch, einzelne Skills, Agenten, Plugins und MCP-Server — wahlweise für die letzten 24 Stunden oder 7 Tage. Version 2.1.175 liefert dazu den neuen Managed-Setting-Schlüssel enforceAvailableModels: Einmal aktiviert, sorgt er dafür, dass der Default-Slot immer innerhalb der Org-Allowlist bleibt und User-Settings die Modellauswahl nicht mehr ausweiten können. Für Teams, die Modell-Konsistenz durchsetzen müssen, ist das eine willkommene Möglichkeit, den Wildwuchs an Session-Konfigurationen zu zähmen.

English

Claude Code 2.1.174 brings a real improvement for VSCode users inside the Account & Usage dialog: the view now breaks down exactly where your token costs are coming from — cache misses, long-context usage, subagents, and most usefully individual skills, agents, plugins, and MCP servers — with a toggle for the last 24 hours or 7 days. Version 2.1.175 adds a new managed settings key, enforceAvailableModels: once enabled, it ensures the Default model slot always stays within the org allowlist and user settings can no longer widen the permitted model set. For teams that need to enforce model consistency, this is a welcome tool for taming the sprawl of per-session configurations.
