English

Developers can review the proposed memory changes before they're applied — no unchecked self-improvement allowed.

Claude Code v2.1.129 · Plugins als .zip aus URL laden
Deutsch

Claude Code hat diese Woche zwei verwandte Plugin-Updates bekommen. Erstens akzeptiert das Plugin-Verzeichnis-Flag jetzt auch gezippte Archive — kein manuelles Entpacken mehr nötig. Zweitens lässt sich mit einem neuen Flag ein Plugin direkt aus einer URL für die aktuelle Session beziehen, ohne es global zu installieren. Perfekt, um Community-Plugins schnell auszuprobieren oder einen frischen Plugin-Build in einem CI-Schritt von einem Bucket zu laden. Die Änderungen kamen mit v2.1.128 und v2.1.129.

English

Claude Code received two closely related plugin updates this week. First, the plugin-directory flag now accepts .zip archives — no manual extraction needed. Second, a new flag lets you fetch a plugin from a URL for the current session only, without installing it globally. Perfect for quickly testing a community plugin or pulling a fresh plugin build from a storage bucket in a CI step. Both changes landed in v2.1.128 and v2.1.129.

BEFEHL · COMMAND
claude --plugin-url <url> [options]
claude --plugin-dir <path-or-zip> [options]
--plugin-url <url>
fetch and load a plugin .zip archive from a URL — active for the current session only
--plugin-dir <path>
load a plugin from a local directory or a .zip archive
# Plugin einmalig aus URL laden · Load a plugin from URL for this session
$ claude --plugin-url https://example.com/my-plugin.zip

# Gezipptes Plugin lokal einbinden · Load a local .zip plugin archive
$ claude --plugin-dir ./my-plugin.zip
