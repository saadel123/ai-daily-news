Claude Code 2.1.178 · Tool(param:value) Permissions
Deutsch

Claude Code 2.1.178 erweitert die Permission-Regeln um eine feinkörnige Parameter-Syntax: Du kannst eine Erlaubnis jetzt direkt an den Eingabewert eines Tools knüpfen — zum Beispiel lässt sich so gezielt nur der Start von Opus-Subagenten blockieren, während andere Subagenten weiter normal laufen. Gleichzeitig laden verschachtelte Skill-Ordner automatisch, wenn du in diesem Unterverzeichnis arbeitest; bei einem Namenskonflikt erscheinen beide Versionen unter eigenen Namen nebeneinander. Im Auto-Modus prüft der Classifier Subagent-Starts jetzt vor dem Launch und schließt eine Lücke, über die ein Subagent bisher ungeprüft blockierte Aktionen starten konnte. Für Teams mit komplexen Subagenten-Workflows ein spürbarer Sicherheitsgewinn.

English

Claude Code 2.1.178 adds a fine-grained parameter syntax to permission rules: you can now tie an allow or deny decision directly to a tool's input value — for example, you can block only Opus subagents while letting all other subagents keep running normally. Nested skills directories now auto-load when you're working inside that subdirectory; on a name clash, both skill versions surface under distinct qualified names side by side. In auto mode, the classifier now screens subagent spawns before launch, closing a gap where a subagent could previously have requested a blocked action without review. A meaningful security improvement for teams running complex multi-subagent workflows.

Von Cursor zu Claude Code · Migration Guide
Deutsch

SpaceX hat Cursor übernommen — ein guter Zeitpunkt, den eigenen Editor-Stack zu überdenken. Claude Code installiert sich mit einem einzigen npm-Befehl und bringt alles mit, was du aus Cursor kennst: regelbasierte Steuerung über CLAUDE.md statt .cursorrules, direkte Terminal-Integration ohne VSCode-Extension und den claude-Befehl als zentralen Einstiegspunkt. Der Wechsel verläuft nahtloser als erwartet — viele .cursorrules-Einträge lassen sich fast eins zu eins in CLAUDE.md überführen. English

SpaceX just acquired Cursor — a good moment to reconsider your editor stack. Claude Code installs with a single npm command and covers everything you know from Cursor: rule-based control via CLAUDE.md instead of .cursorrules, native terminal integration without a VS Code extension, and the claude command as your single entry point. The migration is smoother than you'd expect — most .cursorrules entries translate almost 1:1 into CLAUDE.md. If you relied on Cursor for background agents, also check out the /workflow command — that's Claude Code's answer to long-running background agent tasks.
