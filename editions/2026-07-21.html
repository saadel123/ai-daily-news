Claude Code 2.1.216 · MCP Auto-Background
Deutsch

Claude Code 2.1.216 bringt ein Feature, das lange überfällig war: MCP-Tool-Calls, die länger als zwei Minuten laufen, wandern automatisch in den Hintergrund — die Session friert nicht mehr ein. Die Schwelle lässt sich über eine Env-Var feinjustieren; wer enge Limits mag, stellt 60 000 Millisekunden, wer mehr Spielraum braucht, schiebt den Wert raus. Dazu kommen harte Session-Budgets für WebSearch (Standard: 200 Aufrufe) und Subagent-Spawns — praktisch für CI-Pipelines, die nicht ausufern sollen. Und /ultrareview akzeptiert jetzt PR-Nummern direkt: einfach #216 eintippen, statt die ganze URL zu kopieren.

English

Claude Code 2.1.216 ships a long-awaited quality-of-life fix: MCP tool calls running longer than two minutes now move to background automatically — no more frozen sessions. The threshold is tunable via an environment variable; set it to 60,000 ms for a tighter cutoff or push it up for more headroom. Hard session budgets also land for WebSearch (default 200 calls) and subagent spawns — handy for CI pipelines that shouldn't spiral out of control. And /ultrareview now accepts PR references directly: just type #216 instead of pasting the full URL.

BEFEHL · COMMAND
CLAUDE_CODE_MCP_AUTO_BACKGROUND_MS=<ms> claude [task]
CLAUDE_CODE_MAX_WEB_SEARCHES_PER_SESSION=<n> claude [task]
CLAUDE_CODE_MAX_SUBAGENTS_PER_SESSION=<n> claude [task]
CLAUDE_CODE_MCP_AUTO_BACKGROUND_MS
milliseconds before a long-running MCP call is moved to background (default: 120000)
CLAUDE_CODE_MAX_WEB_SEARCHES_PER_SESSION
hard cap on WebSearch tool calls per session (default: 200)
CLAUDE_CODE_MAX_SUBAGENTS_PER_SESSION
hard cap on subagent spawns per session (default: 200)
# MCP-Calls nach 1 Minute in den Hintergrund · Move MCP calls to background after 1 min
$ CLAUDE_CODE_MCP_AUTO_BACKGROUND_MS=60000 claude task "Index the monorepo"

# Session-Budget für Pipelines deckeln · Cap session budgets in CI pipelines
$ CLAUDE_CODE_MAX_WEB_SEARCHES_PER_SESSION=30 CLAUDE_CODE_MAX_SUBAGENTS_PER_SESSION=10 claude task "..."

💡 Hack des Tages · Tip of the day

Claude Code 2.1.211 · Subagent-Logs im Stream
Deutsch

Wer Claude Code in Pipelines oder CI-Skripten einsetzt und dabei Subagents startet, hat bisher nur das Eltern-Stream-JSON gesehen — die Subagent-Ausgaben blieben verborgen. Mit 2.1.211 lässt sich das mit einem einzigen Flag oder einer Env-Var ändern: Der komplette Text und das Thinking der Subagents fließen ein und landen direkt im stream-json-Output der übergeordneten Session. Besonders nützlich beim Debuggen von Multi-Agent-Workflows, wenn man sehen will, was genau im Untergeschoss passiert.

English

If you run Claude Code in pipelines or CI scripts with subagents, you've only been seeing the parent's stream-JSON — subagent text was hidden. Claude Code 2.1.211 fixes that with a single flag or env var: the full text and thinking from subagents flow directly into the parent session's stream-json output. Especially useful when debugging multi-agent workflows and you want to see exactly what's happening one level down.

BEFEHL · COMMAND
claude [task] --forward-subagent-text
CLAUDE_CODE_FORWARD_SUBAGENT_TEXT=1 claude [task]
--forward-subagent-text
include subagent text and thinking in the parent stream-json output
CLAUDE_CODE_FORWARD_SUBAGENT_TEXT
env-var equivalent; set to 1 to enable persistently
# Subagent-Output im stream-json sichtbar · See subagent output in stream-json
$ claude task "build and test the project" --forward-subagent-text --output-format stream-json

# Dauerhaft aktivieren · Enable persistently via env var
$ export CLAUDE_CODE_FORWARD_SUBAGENT_TEXT=1
