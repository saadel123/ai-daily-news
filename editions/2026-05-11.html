💡 Hack des Tages · Tip of the day

Claude Code 2.1.129 · Plugin per URL laden
Deutsch

Claude Code 2.1.129 bringt ein neues Flag, mit dem du ein Plugin-ZIP direkt aus einer URL für die aktuelle Session laden kannst — ohne es erst lokal zu installieren. Praktisch für Teams, die interne Plugins über einen CI-Job oder ein privates Repository verteilen: einfach die Archiv-URL angeben, und das Plugin ist für die Session aktiv. Das Flag ergänzt die bestehende Option zum Laden aus einem lokalen Verzeichnis, die seit 2.1.128 ebenfalls ZIP-Archive direkt verarbeitet. Ideal, wenn du ein Plugin zwischen CI-Build und Session testen willst, ohne einen manuellen Deploy-Schritt dazwischen.

English

Claude Code 2.1.129 ships a new flag for loading a plugin ZIP archive directly from a URL — no local install needed. Perfect for teams distributing internal plugins via a CI job or private repository: point at the archive URL and the plugin is live for that session. The flag complements the existing local-directory plugin option, which has accepted ZIP archives directly since 2.1.128. Ideal when you want to test a plugin right after a CI build, with no manual deploy step in between.

BEFEHL · COMMAND
claude --plugin-url <url> [options]
--plugin-url <url>
load a plugin .zip archive from a URL for the current session only
--plugin-dir <path>
load a plugin from a local directory or .zip file

$ claude --plugin-url https://tools.company.com/builds/my-plugin.zip

# Lokal entwickeln, dann per URL deployen · Develop locally, then deploy via URL
$ claude --plugin-url https://ci.company.com/artifacts/my-plugin-v1.2.zip
