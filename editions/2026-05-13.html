Codex sicher betreiben · OpenAI's Sandbox-Playbook
Deutsch

Am 8. Mai hat OpenAI einen detaillierten Einblick gegeben, wie Codex intern sicher betrieben wird — und die Erkenntnisse lassen sich direkt auf Claude Code, Cline oder jeden anderen Coding-Agenten übertragen. Das Rezept: striktes Netzwerk-Whitelisting (nur erwartete Ziele erlaubt), fein granulare Sandbox-Grenzen (welche Pfade der Agent schreiben darf), ein Auto-Review-Modus für Low-Risk-Aktionen und unveränderliche Telemetrie-Logs zum Nachvollziehen, was der Agent wirklich getan hat. Wer Coding-Agenten im Team-Setup oder in der CI/CD-Pipeline betreibt, sollte sich dieses Playbook durchlesen — die Kombination aus Sandbox und Audit-Trail ist das Minimum für Produktivsysteme.

English

On May 8th, OpenAI published a detailed look at how they run Codex safely in production — and the lessons map directly onto Claude Code, Cline, or any other coding agent. The recipe: strict network whitelisting (only expected destinations allowed), fine-grained sandbox boundaries (which paths the agent may write), an auto-review mode for low-risk actions, and immutable telemetry logs for auditing what the agent actually did. If you run coding agents in a team environment or CI/CD pipeline, this playbook is worth a close read — sandbox plus audit trail is the minimum for production systems.
