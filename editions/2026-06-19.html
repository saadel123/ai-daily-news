Claude Code 2.1.183 · Auto Mode Safety
Deutsch

Claude Code 2.1.183 legt beim Auto Mode ein Sicherheitsnetz ein: Befehle wie git reset --hard, git checkout -- . und git clean -fd werden jetzt automatisch blockiert, wenn du nicht ausdrücklich darum gebeten hast, lokale Arbeit zu verwerfen. Genauso ist git commit --amend gesperrt, sobald der Commit nicht in dieser Session vom Agenten selbst stammt — das verhindert versehentliches Überschreiben deiner eigenen History. Auch destruktive IaC-Befehle wie terraform destroy, pulumi destroy und cdk destroy sind künftig geschützt, solange du den spezifischen Stack nicht explizit nennst. Als Bonus gibt es mit /config --help einen neuen Slash-Command, der alle Kurzschlüssel der Config auflistet, und das neue Setting attribution.sessionUrl lässt dich den claude.ai-Link aus Commits und Pull Requests weglassen.

English

Claude Code 2.1.183 adds a safety net around auto mode: commands like git reset --hard, git checkout -- . and git clean -fd are now automatically blocked unless you explicitly asked the agent to discard local work. Similarly, git commit --amend is blocked when the commit wasn't made by the agent in the current session — preventing accidental history overwrites. Destructive IaC commands — terraform destroy, pulumi destroy, cdk destroy — are also protected unless you name the specific stack. As a bonus, /config --help is a new slash command that lists all available config shorthand keys, and the new attribution.sessionUrl setting lets you omit the claude.ai session link from commits and PRs in web and Remote Control sessions.

BEFEHL · COMMAND
/config --help
/config key=value
--help
list all available shorthand keys for /config key=value syntax
# Alle Config-Kurzschlüssel anzeigen · List all config shorthand keys
$ /config --help

# Session-Link aus Commits und PRs entfernen · Omit session link from commits and PRs
$ /config attribution.sessionUrl=false
