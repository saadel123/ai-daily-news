💡 Hack des Tages · Tip of the day

Claude Code 2.1.162 · agents --json waitingFor
Deutsch

Seit Claude Code 2.1.162 trägt das JSON-Ausgabeformat für laufende Sessions ein neues Feld, das anzeigt, auf was ein Agent gerade wartet — typischerweise ein offener Berechtigungshinweis. Wer mehrere Hintergrundagenten parallel betreibt, kann damit per Pipe sofort erkennen, welche Session tatsächlich blockiert ist und welche noch läuft. Das lässt sich gut in Monitoring-Skripte einbauen oder in einem einfachen Dashboard zusammenfassen — kein manuelles Durchklicken durch die Agentenliste mehr nötig.

English

Since Claude Code 2.1.162, the JSON output for running sessions includes a new field that shows what an agent is waiting on — typically a pending permission prompt. Anyone running multiple background agents in parallel can now pipe the output to immediately spot which session is blocked and which is still progressing. It's a clean fit for monitoring scripts or a simple dashboard — no more clicking through the agent list manually.

BEFEHL · COMMAND
claude agents --json
# Alle Sessions als JSON · All sessions as JSON
$ claude agents --json

# Blockierte Sessions herausfiltern · Filter only blocked sessions
$ claude agents --json | jq '.[] | select(.waitingFor != null)'
