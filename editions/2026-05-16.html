Claude Code 2.1.142 · Fast Mode auf Opus 4.7
Deutsch

Claude Code 2.1.142 bringt einen Wechsel, der alle Fast-Mode-Nutzer sofort trifft: Statt Opus 4.6 läuft Fast Mode jetzt standardmäßig mit Opus 4.7, das bei schwierigen Engineering-Aufgaben messbar besser ist. Plugins brauchen keine eigene Skills-Ordnerstruktur mehr — eine einzige SKILL.md im Root reicht aus, damit der Marketplace die Fähigkeiten erkennt. Dazu kommen neue Flags für den Agenten-Dashboard-Start, mit denen du Modell, Permission-Mode und Working Directory direkt beim Launch festlegst. Wer mehrere Hintergrund-Sessions gleichzeitig steuert, spart sich damit viele manuelle Konfigurationsschritte.

English

Claude Code 2.1.142 flips a switch that touches every Fast Mode user immediately: instead of Opus 4.6, Fast Mode now runs on Opus 4.7 by default, showing measurable improvements on difficult engineering tasks. Plugins no longer need a dedicated skills folder structure — a single SKILL.md in the root is enough for the marketplace to pick up its skills. On top of that, new flags for the agent dashboard launch let you pin the model, permission mode and working directory right at startup. Anyone juggling multiple background sessions will save a lot of manual setup steps.

BEFEHL · COMMAND
claude agents [options]
--model <name>
set the default model for sessions dispatched from the dashboard
--permission-mode <mode>
set the default permission mode for all dispatched sessions
--effort <level>
set the default effort level for dispatched sessions
--add-dir <path>
add a working directory to the dashboard and all dispatched sessions
# Dashboard mit Opus 4.7 im Auto-Modus starten · Launch dashboard with Opus 4.7 in auto mode
$ claude agents --model claude-opus-4-7 --permission-mode auto

# Sessions auf ein Projektverzeichnis einschränken · Scope sessions to a project directory
$ claude agents --cwd ~/code/my-project

💡 Hack des Tages · Tip of the day

Claude Code 2.1.143 · Plugin-Abhängigkeiten & Kontext-Kosten
Deutsch

Claude Code 2.1.143 macht das Plugin-Ökosystem robuster: Wenn du ein Plugin deaktivieren willst, von dem ein anderes abhängt, verweigert Claude Code das Disable und gibt dir direkt den passenden Befehl, um die ganze Kette abzuschalten. Im Plugin-Marketplace siehst du jetzt außerdem eine Schätzung, wie viel Kontext-Tokens ein Plugin pro Turn und pro Invocation verbraucht — nützlich, bevor du es installierst. Für Repos, bei denen Git-Worktrees schlicht nicht funktionieren, gibt es mit dem neuen Setting worktree.bgIsolation auf none eine saubere Hintertür: Hintergrund-Sessions bearbeiten dann direkt die Working Copy, ohne den EnterWorktree-Schritt.

English

Claude Code 2.1.143 hardens the plugin ecosystem: if you try to disable a plugin that another enabled plugin depends on, Claude Code now refuses and hands you a copy-pasteable command to disable the whole dependency chain. The plugin marketplace now also shows projected context token costs per turn and per invocation — useful to check before you install. For repos where Git worktrees simply don't work, the new worktree.bgIsolation "none" setting is a clean escape hatch: background sessions then edit the working copy directly, skipping the EnterWorktree step.

BEFEHL · COMMAND
claude plugin disable <name>
# Schlägt fehl und zeigt Disable-Kette an · Refuses and prints the disable chain
$ claude plugin disable my-util-plugin

# Worktree-Isolation deaktivieren (settings.json) · Disable worktree isolation
$ claude config set worktree.bgIsolation none
