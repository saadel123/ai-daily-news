Claude Code 2.1.172 · Sub-Agent-Rekursion
Deutsch

Claude Code 2.1.172 bringt eine der interessantesten Neuerungen seit Langem: Sub-Agents können jetzt selbst Sub-Agents spawnen — rekursiv bis zu fünf Ebenen tief. Das eröffnet echten Fan-out für komplexe Aufgaben: ein Orchestrator-Agent kann spezialisierte Teams hochziehen, die ihrerseits Workers starten und Teilaufgaben weiterleiten. Außerdem liest der Bedrock-Connector die AWS-Region jetzt direkt aus den lokalen Config-Dateien — kein manuelles Setzen von AWS_REGION mehr nötig. Kleines, aber feines Extra: der Plugin-Marketplace hat jetzt eine Suchleiste, damit du bei 200+ Plugins nicht mehr endlos scrollst.

English

Claude Code 2.1.172 lands one of the most interesting changes in a while: sub-agents can now spawn their own sub-agents, recursively up to five levels deep. That unlocks real fan-out for complex jobs — an orchestrator agent can spin up specialized teams that themselves launch workers and pass sub-tasks down the chain. Bedrock also now reads the AWS region from local config files, matching AWS SDK precedence — no more manually setting AWS_REGION. Small but welcome bonus: the plugin marketplace now has a search bar, so you don't have to scroll endlessly through 200+ plugins.

💡 Hack des Tages · Tip of the day

Claude Code 2.1.163 · /plugin list · Version-Enforcement
Deutsch

Claude Code 2.1.163 bringt zwei praktische Neuerungen für Teams mit Enterprise-Setup. Erstens gibt es jetzt einen Slash-Command, mit dem du auf einen Blick siehst, welche Plugins in deiner Session aktiv oder deaktiviert sind — unerlässlich, wenn du mehrere Plugins verwaltest und den Überblick behalten willst. Zweitens können Admins mit neuen Managed Settings eine Mindest- und Höchstversion für Claude Code vorschreiben: wer mit einer Version außerhalb des erlaubten Bereichs startet, wird direkt auf die zugelassene Version hingewiesen. Das ist vor allem in CI-Pipelines und Unternehmensumgebungen Gold wert, wo Versionsdisziplin zählt.

English

Claude Code 2.1.163 adds two practical features for teams running enterprise setups. First, there's a new slash command that shows at a glance which plugins are active or disabled in your session — essential when you're juggling multiple plugins and need to stay on top of what's loaded. Second, admins can now use new managed settings to mandate a minimum and maximum Claude Code version: anyone starting outside the allowed range gets pointed straight to an approved version. Especially valuable in CI pipelines and corporate environments where version discipline matters.
